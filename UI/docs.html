Docs chevron_right Introduction

Mesa Protocol

Mesa is an open-source orchestration layer for embedded finance workflows. It resides directly between your application logic and the Stellar ledger, managing multi-step transactions, retries, and network delays so you don't have to write custom queue workers or state recovery code.

replay

Durable Retries

Horizon timeouts or ledger congestion automatically trigger retries with configurable exponential backoff.

save

Persistent State

Every step of your workflow is checkpointed in a database, ensuring safety across server reboots.

Docs chevron_right Core Concepts

Core Architecture

Mesa separates the *definition* of a workflow from its *execution*.

1. Workflow Definition

Created using the SDK's fluent `FlowBuilder` API. Workflow definitions are saved to Postgres as JSON files. They contain steps (e.g. Receive, Delay, Payment) and their target parameters.

2. Steps & Providers

Each step specifies a **Provider** that executes the action. For instance, the `stellar` provider handles payment validation and submissions, while the `delay` provider handles scheduler wait queues.

Docs chevron_right Stellar Integration

Connecting to Stellar

Mesa resolves private keys at execution time using reference tokens, ensuring maximum key security on server environments.

index.ts 
import { Mesa } from '@mesaprotocol/sdk';

// Initialize Mesa
Mesa.configure({ runtimeUrl: 'http://localhost:3001' });

const flow = Mesa.flow('hello-world-flow')
  .receive({
    asset: 'XLM',
    minAmount: 25,
    toAddress: 'GD3ZJ3A4...',
  })
  .delay({ seconds: 5 })
  .payment({
    horizonUrl: 'https://horizon-testnet.stellar.org',
    senderSecretRef: 'SENDER_SECRET',
    to: 'GA4UFV...',
    amount: 25,
  })
  .build();

await Mesa.execute(flow);
